Эти задачи помогут не только создать функциональную базу данных, но и повысить общую эффективность работы риэлторского агентства.

2. Клиент (Client)
   - ID (уникальный идентификатор)
   - Имя

5. Информация о рынке (Market Information)
   - ID (уникальный идентификатор)
   - Регион

   - Статистика (количество продаж, аренды)
   - Тенденции (рост/падение цен)
## 2.3. Определение отношений и мощности отношений между объектами.
1. Объект недвижимости – Сделка
   - Отношение: Один объект может участвовать в нескольких сделках (1:N)
   - Мощность: 1 (один объект) к N (несколько сделок)

2. Клиент – Сделка
   - Отношение: Один клиент может проводить несколько сделок (1:N)
   - Мощность: 1 (один клиент) к N (несколько сделок)

3. Клиент (покупатель) – Риэлтор
   - Отношение: Один риэлтор может работать с несколькими клиентами (1:N)
   - Мощность: 1 (один риэлтор) к N (несколько клиентов)

4. Объект недвижимости – Риэлтор
   - Отношение: Один риэлтор может представлять несколько объектов (1:N)
   - Мощность: 1 (один риэлтор) к N (несколько объектов)

5. Информация о рынке – Объект недвижимости
   - Отношение: Один объект недвижимости связан с одной записью информации о рынке (1:1)
   - Мощность: 1 (один объект) к 1 (одна информация о рынке)
## 2.4. Построение концептуальной модели.
Концептуальная модель представляется в виде диаграммы, которая показывает сущности, их атрибуты, а также отношения между ними. В текстовом формате это может быть описано следующим образом:
[Client] --(1:N)-- [Transaction] --(N:1)-- [Property]
        \                |                     |
         \               |                     |
          (1:N)         |                     |
           |            |                     |
        [Realtor] ---- (1:N) ------------ [Market Information]
- Client связан с Transaction (один клиент может участвовать в нескольких сделках).
- Transaction связывает Client и Property (каждая сделка включает одного клиента и один объект недвижимости).
- Realtor имеет отношение 1:N с Client и Property (один риэлтор может работать с несколькими клиентами и объектами).
- Market Information можно считать справочной таблицей, предоставляющей данные для объектов недвижимости.

Эта концептуальная модель позволяет структурировать базу данных, а также сформировать основу для разработки реляционной модели и дальнейшего проектирования системы.
